# Naming

- C# conventions: PascalCase for types, methods, properties, constants; camelCase for locals and parameters; `_camelCase` for private fields.
- Interfaces start with `I`; async methods end with `Async`.
- Names should reveal intent — avoid abbreviations, single letters (except trivial loop indices), and vague names like `data`, `temp`, `mgr`, `obj` or single char names.
- Booleans read as predicates (`isEnabled`, `hasItems`, `canRetry`).
- Avoid Hungarian notation and type names embedded in identifiers.
- Be consistent with the surrounding code's existing vocabulary.
